# Проєкт: GoIT «Бази даних» (ІПЗ_4 семестр)

## Середовище

- **СУБД:** MariaDB 12.2 (drop-in заміна MySQL, повністю сумісна з курсом — MySQL Workbench розпізнає як MySQL-сервер).
- **Підключення:** `mariadb -u vlad -pvlad` (або через Workbench: 127.0.0.1:3306, user=`vlad`, pass=`vlad`).
- **БД для всіх практичних:** `publishing`.

## Робочий процес для практичних

**ОБОВ'ЯЗКОВО:** усі SQL-запити, які виконуються в практичних, мають бути збережені у файли під `sql/`. **Тільки скріншоти результатів недостатньо** — нам треба мати повноцінний текстовий код, який можна редагувати/повторно запускати/комітити.

Структура папки `sql/`:

```
sql/
├── pr04_ddl_dml.sql          # схема + INSERT для теми 4
├── pr05_dql.sql              # 19 задач теми 5
├── pr06_triggers.sql         # тригери теми 6
├── pr07_subqueries_cte.sql   # підзапити, CTE
├── pr08_functions.sql        # вбудовані функції
└── pr09_python_export.sql    # SQL, що використовується в Python (тема 9)
```

Усередині кожного файлу:
- Розділяти задачі коментарями `-- ===== Задача N. <назва> =====`.
- Включати **очікуваний результат** як коментар або через `-- Result: ...` блок.
- Зберігати **актуальний код**, який запускався в Workbench (не варіанти, що не пройшли).

Скрінити в `screens/`, але це додаткове (для зошита), а не заміна SQL-файлів.
